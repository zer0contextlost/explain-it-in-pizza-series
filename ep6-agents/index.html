♻️ The ReAct Loop: Think → Act → Observe → Repeat

Pizza metaphor: The chef doesn't know the full recipe. He thinks, tries, observes, and adjusts.

How It Works

Start the order and watch the agent cycle through THINK (decide what to do), ACT (attempt it), and OBSERVE (see what happened). This loop repeats until the pizza is complete. Use Step Mode to control it manually.

🔧 Tool Calling: Extending the Chef's Hands

Pizza metaphor: The chef can't grow tomatoes. But he can call the supplier, use the oven, or ring up the bill.

How It Works

Tools are the chef's interface to the world. Each tool has a purpose. Try disabling tools to see how the chef's plans adapt or fail. Tools are how agents do more than think.

📝 Planning: The Whiteboard Before Action

Pizza metaphor: The chef writes a numbered checklist before touching the dough.

How It Works

Good planning is slower upfront but faster overall. Watch the chef plan each order with numbered steps, then execute them. Compare with the chaotic "no planning" mode to see why structure matters.

📝 Short-Term Memory: The Notepad in the Apron

Pizza metaphor: Notes in the chef's apron pocket. Forgotten when the shift ends.

How It Works

The context window is limited. When new facts arrive, old ones fall off the bottom of the notepad. This is why agents struggle with long conversations — they forget what was said at the start.

📖 Long-Term Memory: The Leather-Bound Regulars Book

Pizza metaphor: Customer records stored under the counter. Persists forever.

How It Works

Save important facts to long-term memory and they survive context shifts. Customers return after a new shift, and the chef remembers their preferences. This is external memory — databases, vector stores, knowledge graphs.

👥 Multi-Agent Systems: The Whole Brigade

Pizza metaphor: Different chefs specializing in different roles.

How It Works

Sous Chef plans. Dough Master stretches. Topping Specialist garnishes. Oven Tender bakes. Order ticket flows through the brigade. When one station is disabled, the pipeline breaks. This is why specialized agents are powerful at scale.

🔁 Infinite Loops: Chef in a Broken Record

Pizza metaphor: Chef thinks "I need cheese" → opens fridge → forgets why → thinks "I need cheese" → opens fridge...

How It Works

When the agent gets stuck, it can loop forever. Set a max-steps limit to catch this. Without circuit breakers, the model can waste tokens and spin wheels indefinitely.

👻 Hallucinated Tools: Confident in Fake Capabilities

Pizza metaphor: Chef confidently picks up an imaginary truffle injector and swears it worked.

How It Works

The most dangerous failure is when the model hallucinates a tool that doesn't exist, calls it, fabricates a result, and reports success. Enable tool validation to catch this invisible mistake.
